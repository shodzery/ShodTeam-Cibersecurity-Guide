# Code of Conduct

## Purpose

This project is authored and maintained by [Shodzery](https://github.com/shodzery) and [ShodTeam](https://github.com/ShodTeam). It is a technical reference maintained by and for security professionals. The purpose of this Code of Conduct is to define the standards of behavior that allow this community to function effectively, maintain technical quality, and remain a resource that practitioners can rely on.

## Scope

This Code of Conduct applies to all spaces associated with this project: the repository, issue tracker, pull requests, discussions, and any communication that occurs in the context of this project.

## Standards of Acceptable Behavior

Participants are expected to:

- Engage in technical discussion with precision and intellectual honesty
- Accept technically accurate criticism of their contributions without personal defensiveness
- Acknowledge mistakes and correct them openly
- Distinguish clearly between established fact, accepted practice, contested opinion, and personal preference
- Give credit to original authors and sources
- Maintain the expectation that all content is read by practitioners who rely on its accuracy

## Standards of Unacceptable Behavior

The following behaviors are not acceptable:

- Personal attacks, harassment, or discriminatory language of any kind
- Deliberate misrepresentation of technical facts
- Submission of content designed to cause harm, facilitate unauthorized access, or violate applicable laws
- Plagiarism or submission of others' work without attribution
- Repeated low-quality contributions that consume maintainer time without value
- Promotion of products or services without relevant technical disclosure
- Intimidation or persistent unwelcome contact

## Ethical Framework for Security Content

This repository contains documentation of both offensive and defensive security techniques. Contributors and users are expected to operate within the following ethical boundaries:

- Offensive techniques documented here are intended for authorized security testing, research, and education only
- No content is intended to facilitate attacks against systems without explicit authorization from the system owner
- Documentation of attack techniques must include defensive countermeasures where applicable
- Vulnerability research documented here must comply with responsible disclosure principles

The presence of offensive techniques in this repository does not constitute authorization to use those techniques against any system. Contributors who submit content that appears designed to facilitate unauthorized access will have their contributions rejected and may be banned from the project.

## Enforcement

Maintainers are responsible for enforcing this Code of Conduct. The project is maintained by [Shodzery](https://github.com/shodzery) and [ShodTeam](https://github.com/ShodTeam). Reports of violations should be submitted via the project's issue tracker using the appropriate template, or directly to the maintainers.

Upon receiving a report, maintainers will:

1. Acknowledge receipt within 48 hours
2. Investigate the report
3. Take appropriate action, which may include: warning the individual, removing the offending content, restricting participation, or permanently banning the individual from the project

Maintainers who violate this Code of Conduct are subject to the same enforcement process, administered by other maintainers.
